# Admin Panel Workflow

Этот документ нужен не для разработки, а для ежедневной работы с контентом в `/ru/admin`.

## Главная идея

Админка сейчас лучше всего работает как контентный пульт в таком порядке:

1. Сначала смотри `Content board`.
2. Потом переходи в `Реестр навыков`.
3. Потом открывай `Банк задач` только для конкретного `skill_id`.
4. После правок возвращайся в board и смотри, исчез ли дефицит.

Если идти в обратном порядке, легко потеряться в задачах без понимания, какой навык реально важнее добивать.

## Как читать админку

### 1. Content board

Это главный экран для ежедневной работы.

Что он показывает:

- `Навыков к доработке`: сколько skill сейчас требуют действия.
- `Навыков без задач`: пустые skill, где банк еще не начат.
- `Дефициты`: skill не проходят ready gate.
- `Без summary`: skill не имеют краткого описания.
- `Без trainer`: skill не имеют trainer link.

Блок `Очередь действий`:

- показывает самые проблемные skill сверху;
- кнопка `Открыть задачи` переводит сразу к task bank по нужному `topicId/skillId`;
- badges под skill показывают, чего именно не хватает.

Блок `Карта покрытия`:

- группирует тему по `branch`;
- помогает видеть, где провал не в одной задаче, а во всей ветке;
- если в ветке много `Требуют действия`, значит ее лучше добивать серией, а не прыгать по разным темам.

### 2. Реестр контента

Используй его как обзор темы целиком:

- сколько skill уже имеют задачи;
- сколько задач всего по теме;
- есть ли `routes`;
- есть ли `prereq edges`;
- есть ли warning по конфигурации темы.

Это не основной рабочий экран для редактирования, а скорее контрольная панель темы.

### 3. Реестр навыков

Используй его для настройки самого skill:

- `title`
- `summary`
- `kind`
- `status`

Рабочее правило:

- если у skill нет `summary`, сначала заполни summary;
- если skill еще сырой, не спеши ставить `ready`;
- если нужен только контентный фикс, не трогай override без причины.

### 4. Банк задач

Это уже точечная работа по одному skill.

Лучший режим:

1. Выбери `topic`.
2. Выбери `skill`.
3. Работай только внутри него.

Не пытайся редактировать задачи “по всей теме сразу”. Так быстро теряется контекст качества skill.

## Рекомендуемый ежедневный цикл

### Режим 1: доведение продукта до ready

1. Открой `Content board`.
2. Возьми верхний skill из `Очереди действий`.
3. Если нет `summary`, сначала опиши skill.
4. Если нет `trainer`, реши, нужен ли ему trainer entry сейчас.
5. Если не хватает задач, открой `Банк задач`.
6. Добей coverage по bands.
7. Вернись в board и проверь, ушел ли skill из очереди.

### Режим 2: заполнение новой ветки

1. Выбери одну тему.
2. Посмотри в `Карта покрытия`, какая `branch` пустая или слабая.
3. Работай только по этой ветке до приемлемой плотности.
4. Не распыляйся на 5 разных branch в один проход.

### Режим 3: финальная полировка темы

Проверь по теме:

- нет warning;
- у skill есть summary;
- у ready-skill есть достаточное покрытие задач;
- нет явных пустых branch;
- routes и prereq не выглядят забытыми.

## Как расставлять приоритеты

Лучший порядок сейчас такой:

1. `skill без задач`
2. `skill не проходит ready gate`
3. `skill без summary`
4. `skill без trainer`
5. только потом косметические правки

Почему так:

- пустой skill ломает целостность продукта сильнее всего;
- потом идет качество покрытия;
- потом уже удобство и навигация.

## Что считать “достаточно готовым”

Для себя можно держать такой practical DoD:

- у skill есть понятный `title`;
- есть короткий `summary`;
- есть нормальный набор задач;
- coverage не провален по bands;
- skill не выглядит случайным или дублирующим соседний;
- статус `ready` выставлен осознанно, а не “потому что задач уже много”.

## Частые сценарии

### Если board пишет `нет задач`

- открой `Открыть задачи`;
- создай стартовый набор задач именно для этого skill;
- после сохранения обнови админку.

### Если board пишет `не проходит ready gate`

- обычно не хватает одной или нескольких difficulty bands;
- смотри покрытие вида `A/B/C` прямо в карточке;
- добивай именно недостающий band, а не просто добавляй еще случайные задачи.

### Если board пишет `нет summary`

- правь это в `Реестр навыков`;
- summary должен коротко объяснять, что именно умеет ученик после этого skill.

### Если board пишет `нет trainer link`

- это не всегда блокер для контента;
- сначала закрывай задачи и summary;
- trainer link имеет смысл добивать после того, как skill уже содержательно оформлен.

## Практическое правило

Если сомневаешься, что делать следующим:

- не открывай новую тему;
- не редактируй все подряд;
- бери первый skill из `Очереди действий` и доводи его до внятного состояния.

Это сейчас самый быстрый способ сделать продукт визуально и содержательно цельным.
